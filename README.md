# zoo-OOP
Задачи
1. Создайте базовый класс Animal
Атрибуты (сделайте приватными, где уместно):

name (имя)

age (возраст)

__health_status (статус здоровья: healthy - True или sick - False) — инкапсулируйте.

Методы:

get_info() — возвращает строку с именем, возрастом и здоровьем.

make_sound() — возвращает строку со звуком животного (пока просто "Some animal sound").

feed() — кормит животное: если статус здоровья "sick", меняет на "healthy" (выздоравливает). Выведите сообщение об этом.

Геттер и сеттер для __health_status (через @property и @health_status.setter).

2. Создайте дочерние классы
Mammal (млекопитающее)

Добавляет атрибут fur_color (цвет шерсти).

Переопределите get_info() так, чтобы он включал цвет шерсти.

Переопределите make_sound() — например, "Roar" или "Bark".

Bird (птица)

Добавляет атрибут wing_span (размах крыльев в см).

Переопределите get_info() — добавить размах крыльев.

Переопределите make_sound() — "Chirp" или "Squawk".

Reptile (рептилия)

Добавляет атрибут is_venomous (ядовитая ли — булево значение).

Переопределите get_info() — указать, ядовитая или нет.

Переопределите make_sound() — "Hiss".

3. Создайте класс Zoo
Приватный атрибут __animals — список, хранящий всех животных (инкапсуляция).

Методы:

add_animal(animal) — добавить животное (любого типа). Проверьте, что передан именно объект Animal или его потомок. Если нет — выбросить исключение или вывести предупреждение.

remove_animal(name) — удалить животное по имени.

find_animals_by_type(class_name) — найти всех животных, являющихся экземплярами переданного класса (например, "Mammal", "Bird"). Вернуть список.

get_all_animals_info() — полиморфный метод: вызывает get_info() для каждого животного в списке и возвращает строку со всей информацией (или печатает).

feed_all_animals() — вызывает метод feed() для каждого животного в зоопарке.

get_sick_animals() — возвращает список животных, у которых health_status == "sick".

Советы по выполнению
Инкапсуляция

Статус здоровья сделайте приватным (__health_status). Используйте @property для доступа к нему извне.

Список животных внутри Zoo тоже должен быть приватным (__animals). Не возвращайте его напрямую из методов — лучше возвращайте копию или итерируйте внутри класса.

Наследование

В конструкторах дочерних классов обязательно вызывайте super().__init__(name, age) для инициализации общих атрибутов.

Добавляйте специфичные атрибуты (fur_color, wing_span, is_venomous) после вызова super().

Полиморфизм

В Zoo.get_all_animals_info() просто проходите по списку и вызывайте animal.get_info() для каждого. Python сам подставит нужную версию метода в зависимости от типа животного.

Аналогично с feed_all_animals() — у каждого животного есть свой feed(), но вызов одинаков.

Проверки и исключения

В add_animal() можно использовать isinstance(animal, Animal), чтобы убедиться, что добавляется корректный объект.

В remove_animal() учтите, что может не найтись животное с таким именем — верните False или выбросите исключение.

Дополнительная практика (по желанию)

Реализуйте метод transfer_animal(zoo, animal_name) для перемещения животного из одного зоопарка в другой.
